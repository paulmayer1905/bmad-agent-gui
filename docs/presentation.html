01

Vue d'ensemble

BMAD Agent GUI est une application desktop cross-platform construite avec Electron et React. Elle offre une interface graphique complète pour interagir avec les dix agents spécialisés de la méthode BMAD (Breakthrough Method for Agile Development), directement depuis votre ordinateur, sans passer par un navigateur.

L'application permet de passer des instructions textuelles à des agents IA — chacun expert dans un domaine précis du développement logiciel — et d'obtenir des réponses riches, structurées et sauvegardées automatiquement sous forme de documents.

Projets simultanés
💡 La méthode BMAD est une approche structurée du développement logiciel assisté par IA. Chaque agent correspond à un rôle dans une équipe projet réelle : analyste, architecte, développeur, PM, QA… BMAD Agent GUI rend cette équipe virtuelle opérationnelle en quelques secondes.

Problème résolu

Les outils LLM génériques (ChatGPT, Claude, etc.) manquent de contexte métier, de mémoire de projet persistante, et de rôles définis. Basculer entre différents chats pour différentes tâches fait perdre le fil du projet. BMAD Agent GUI centralise tout : contexte partagé, agents spécialisés, historique documentaire structuré, et coordination inter-agents.

03

Les 10 agents BMAD

Chaque agent est un LLM configuré avec un system prompt spécialisé, des instructions de rôle précises, et un accès au contexte partagé du projet. Ils peuvent travailler seuls ou être coordonnés en pipeline.

📊 Analyste
Business Analysis

Recueille et structure les besoins, rédige les user stories, identifie les risques et parties prenantes. Produit des documents de cadrage complets.

🏗️ Architecte
Software Architecture

Conçoit l'architecture système, choisit les patterns, définit les interfaces et les décisions techniques structurantes.

📋 Product Manager
Product Management

Définit la vision produit, priorise le backlog, rédige les PRD et aligne les équipes sur les objectifs business.

📦 Product Owner
Product Ownership

Gère le backlog sprint par sprint, affine les stories, valide les critères d'acceptation et les DoD.

🎨 UX Expert
User Experience

Conçoit les parcours utilisateurs, wireframes, spécifications d'interface et recommandations d'accessibilité.

💻 Développeur
Software Development

Implémente les fonctionnalités, revoit le code, propose des refactorings et explique les choix techniques.

🐛 QA Engineer
Quality Assurance

Rédige les plans de test, identifie les cas limites, analyse les bugs et propose des stratégies de test.

🏃 Scrum Master
Agile Coaching

Facilite les cérémonies Scrum, lève les blocages, mesure la vélocité et accompagne l'amélioration continue.

🧙 BMAD Master
Master Coordinator

Supervise l'ensemble du projet, coordonne les agents et prend les décisions transverses de haut niveau.

🎭 Orchestrateur
Pipeline Orchestration

Organise et exécute des pipelines multi-agents, délègue les tâches et consolide les résultats en un livrable cohérent.

04

Fonctionnalités principales

💬 Chat multi-agents

Conversation directe avec n'importe quel agent. Historique de session, streaming des réponses en temps réel, contexte projet injecté automatiquement.

🎉 Party Mode (collaboration)

Réunissez plusieurs agents en simultané. Chacun réagit à la discussion, vous pouvez interpeller un agent spécifique ou laisser l'Orchestrateur coordonner.

⚡ Pipelines automatisés

Définissez des séquences d'agents à exécuter avec des prompts préconfigurés. Suivi visuel étape par étape avec le PipelineVisualizer animé.

📁 Workspace réel

Pointez l'application sur votre dossier projet. Les agents lisent vos fichiers source (code, docs, configs) pour fournir des réponses contextualisées.

📚 Historique documentaire

Chaque réponse agent peut être sauvegardée en Markdown. Navigateur 3 colonnes pour parcourir projets, dossiers par agent, et documents.

📦 Export ZIP

Exportez tout le projet documentaire en archive ZIP en un clic. Idéal pour partager avec l'équipe ou archiver une version.

🔍 Command Palette

Accès rapide à toute la navigation via Ctrl+K. Recherche fuzzy sur 12 commandes, navigation clavier complète.

🔔 Notifications OS

Quand un pipeline se termine, l'OS envoie une notification native — vous pouvez travailler sur autre chose en attendant.

🧠 Mémoire partagée

Le contexte projet (objectif, stack, décisions) est partagé entre tous les agents. Cohérence garantie sur toute la session.

🌐 Multi-fournisseur LLM

Ollama (local/gratuit), Claude d'Anthropic, Gemini de Google, et OpenAI — configurables par agent ou globalement.

05

Interface utilisateur

Structure de navigation

L'interface est divisée en une sidebar fixe à gauche et un espace de travail à droite. La sidebar donne accès à toutes les sections principales :

🏠 Tableau de bord

Vue synthétique de l'état du projet : agents actifs, dernières conversations, métriques d'utilisation (tokens, coûts estimés), raccourcis rapides vers les actions fréquentes.

🤖 Agents

Liste des 10 agents avec leur statut, leur modèle LLM configuré, et les statistiques de session. Lancement direct d'une conversation depuis cette vue.

💬 Conversations

Historique complet de toutes les sessions par agent. Reprise d'une conversation existante, recherche par contenu, suppression de sessions.

🎉 Party Chat

Mode collaboration multi-agents. Interface de chat avec sélection des agents participants, PipelineVisualizer pour les exécutions séquentielles, et mode Party libre.

📁 Workspace

Explorateur de fichiers du projet réel. Navigation dans l'arborescence, lecture de fichiers, injection dans le prompt courant d'un agent.

📚 Historique documentaire

Navigateur 3 colonnes. Panneaux : projets doc (gauche), dossiers par agent (centre), liste des documents (droite). Ouverture directe dans le MarkdownViewer.

⚙️ Paramètres

Configuration des clés API, sélection des modèles LLM par agent, paramètres de température, URL Ollama, préférences d'interface.

Composants UI notables

⌨️ CommandPalette

Overlay Ctrl+K, fuzzy search, navigation flèches/Enter/Escape. S'ouvre depuis n'importe quelle page.

📊 PipelineVisualizer

Tracker animé d'étapes de pipeline. Mode compact (dots) ou complet (timeline). Spinner pour l'état running.

📝 MarkdownViewer

Panneau slide-in depuis la droite. Mode lecture (rendu HTML) + mode édition (textarea). Sauvegarde Ctrl+S.

Design system

  • Thème sombre — fond #0F0F1A, accents violet (#6C63FF) et vert-cyan (#00D4AA)
  • Animations fluides — pageSlideIn à chaque navigation, cmdFadeIn pour la palette, mdSlideIn pour le viewer
  • Variables CSS — toutes les couleurs, espacements et rayons centralisés en CSS custom properties
  • Responsive — layout adaptatif avec grilles CSS, scroll géré par zone
  • Feedback utilisateur — press animation sur tous les boutons, focus rings accessibles, états de chargement animés
06

Cas d'usage détaillés

1

Démarrer un nouveau projet logiciel

Contexte : Une équipe reçoit un brief client. Ils veulent structurer le projet en 2 heures au lieu de 2 jours.

  • 1Ouvrir BMAD GUI, créer un nouveau workspace, pointer sur le dossier projet
  • 2Aller dans Party Chat, sélectionner Analyste + PM + Architecte
  • 3Décrire le brief en langage naturel : "Construire une appli de réservation de salles pour 500 utilisateurs…"
  • 4L'Analyste extrait les besoins → le PM rédige la vision → l'Architecte propose une stack
  • 5Chaque document généré est sauvegardé automatiquement dans l'historique documentaire
  • 6Exporter le dossier ZIP pour partager avec les parties prenantes
2

Pipeline de spécification automatisé

Contexte : Un développeur solo veut produire des specs complètes pour une feature en une commande.

  • 1Dans Party Chat, créer un pipeline : Analyste → PM → PO → UX Expert
  • 2Entrer le titre de la feature : "Système de notifications push multi-canal"
  • 3Lancer le pipeline — le PipelineVisualizer affiche la progression en temps réel
  • 4Chaque agent s'appuie sur la sortie du précédent (contexte partagé automatiquement)
  • 5Une notification OS s'affiche quand tout est terminé
  • 64 documents structurés prêts en ~5 minutes
3

Revue de code intelligente

Contexte : Un développeur veut obtenir une revue de code approfondie enrichie du contexte architectural.

  • 1Pointer le workspace sur le dossier du projet (src/, tests/)
  • 2Ouvrir l'agent Développeur en conversation directe
  • 3Dans l'onglet Workspace, sélectionner les fichiers à revoir et les injecter dans le prompt
  • 4L'agent lit le code avec le contexte architectural de l'Architecte déjà en mémoire
  • 5Suivre avec l'agent QA pour des suggestions de tests
  • 6Sauvegarder le rapport de revue en Markdown dans l'historique
4

Planification d'un sprint

Contexte : L'équipe prépare le sprint planning pour la semaine suivante.

  • 1Ouvrir le Scrum Master avec le backlog en contexte (fichier JIRA export ou Markdown)
  • 2Demander : "Propose un sprint de 10 points basé sur nos capacités actuelles"
  • 3Passer au Product Owner pour affiner les critères d'acceptation des stories sélectionnées
  • 4Utiliser le Développeur pour estimer la complexité technique de chaque story
  • 5Document de sprint plan sauvegardé et exportable
5

Documentation continue d'un projet existant

Contexte : Un projet legacy sans documentation. Besoin de produire rapidement une doc technique.

  • 1Créer un nouveau projet documentaire dans BMAD GUI
  • 2Pointer le workspace sur le projet legacy
  • 3L'Architecte analyse le code source et génère un document d'architecture existante
  • 4Le Développeur documente les modules critiques en parcourant les fichiers injectés
  • 5Le QA identifie les zones sans couverture de test
  • 6Naviguer dans l'Historique documentaire, éditer chaque doc via MarkdownViewer, exporter ZIP
07

Gestion documentaire

BMAD Agent GUI intègre un système de documentation automatique : chaque réponse significative d'un agent peut être persistée sous forme de fichier Markdown, organisée par projet et par agent.

Structure des projets documentaires

# ~/.bmad/doc-projects/ mon-projet/ ├── meta.json # Métadonnées, compteur de docs ├── 01-analyse/ │ ├── 20260304-093012-analyse-besoins.md │ └── 20260304-141500-user-stories.md ├── 02-architecture/ │ └── 20260304-100200-architecture-systeme.md ├── 06-developpement/ │ └── 20260304-154200-revue-code-auth.md └── _historique/ └── 20260304-160000-session-recap.md

Fonctionnalités du gestionnaire documentaire

  • Création de projets multiples — Séparez vos projets clients, produits internes, ou expérimentations
  • Sélection active — Le projet actif est persisté au redémarrage de l'application, restauré automatiquement
  • Arborescence par agent — Les documents sont classés dans un dossier par agent générant la sortie
  • Horodatage automatique — Chaque fichier porte un timestamp YYYYMMDD-HHMMSS pour l'ordre chronologique
  • Édition inline — Le MarkdownViewer permet de modifier un doc sans quitter l'application
  • Export ZIP — Archive l'intégralité d'un projet documentaire, prête à partager ou stocker
  • Protection anti-traversal — Les chemins de fichiers sont validés pour éviter toute écriture hors du projet

MarkdownViewer

👁️ Mode lecture

Rendu HTML complet du Markdown : titres, listes, code blocks, tableaux, emphases. Police monospace pour les blocs de code.

✏️ Mode édition

Textarea plein écran pour modifier le contenu. Sauvegarde avec Ctrl+S. Bascule lecture/édition sans fermer le panneau.

Configuration & déploiement

Prérequis

  • Node.js 18+ — runtime JavaScript côté Electron/backend
  • Git — pour cloner le projet
  • Au moins un fournisseur LLM configuré (clé API ou Ollama local)

Installation rapide

# Cloner et installer git clone https://github.com/paulmayer1905/bmad-agent-gui cd bmad-agent-gui npm install # Lancer en mode développement npm run electron:dev # Build production (Windows .exe) npm run electron:build

Configuration LLM

Les paramètres LLM sont stockés dans ~/.bmad/ai-config.json. Ils sont modifiables depuis l'interface Paramètres :

🦙 Ollama (local, gratuit)

Configurer l'URL Ollama (défaut : http://localhost:11434) et le modèle (llama3.1, mixtral, etc.). Aucune clé API requise.

🧠 Anthropic Claude

Entrer la clé API Anthropic. Modèles : claude-opus-4, claude-sonnet-4, claude-haiku-3. Streaming natif.

🌟 Google Gemini

Clé API Google AI Studio. Modèles : gemini-2.0-flash, gemini-1.5-pro. Support streaming.

🤖 OpenAI

Clé API OpenAI. Compatible GPT-4o, GPT-4-turbo, o1-mini. URL configurable (compatible Azure OpenAI).

Persistance des données

Toutes les données utilisateur sont stockées localement dans ~/.bmad/ :

~/.bmad/ ├── ai-config.json # Clés API, modèles, paramètres ├── gui-state.json # Dernier projet actif, préférences ├── project-context.json # Mémoire partagée des agents ├── sessions/ # Historique des conversations ├── doc-projects/ # Documents générés └── workspaces/ # Configurations des workspaces

Suites de tests

🤖 agent-coordinator.test.js

Tests de coordination multi-agents, pipelines, délégation et party mode. Vérifie l'ordre d'exécution et la transmission de contexte.

📚 documentation-manager.test.js

Tests CRUD projets doc, génération d'arbre, écriture sécurisée de fichiers, protection anti-traversal, export ZIP.

📊 pipeline-quality.test.js

Tests de qualité des pipelines : gestion d'erreurs, timeouts, reprise sur échec, métriques de performance.

💬 system-prompts.test.js

Validation des system prompts de chaque agent : format, cohérence, injection de contexte, longueur maximale.

📁 workspace-extended.test.js

Tests étendus du gestionnaire de workspace : lecture fichiers, filtrage, injection dans prompts, limites de taille.

🔧 Unit tests

Tests unitaires des modules individuels : AIService, ProjectContext, WorkspaceManager, BMADBackend.

🔒 Les tests incluent des vérifications de sécurité : protection contre les path traversal attacks dans le gestionnaire documentaire, validation des entrées, et isolation des modules pour prévenir les effets de bord.

Lancer les tests

# Tous les tests node node_modules/jest/bin/jest.js --no-cache --forceExit # Suite spécifique node node_modules/jest/bin/jest.js documentation-manager --forceExit # Mode watch (développement) node node_modules/jest/bin/jest.js --watch
